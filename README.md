# מנהל המשימות שלי 🤖

מערכת לניהול משימות יומיומיות דרך צ'אט בשפה חופשית, במקום ממשק מסורתי. מבוססת על Function Calling עם Gemini (Google AI Studio).

## איך זה עובד

- **`todo_service.py`** — שכבת הנתונים. משימות נשמרות במערך בזיכרון (בלי מסד נתונים, כי זה דמו), עם השדות: קוד, כותרת, תיאור, סוג, תאריך התחלה, תאריך סיום, סטטוס. חושף ארבע פונקציות: `get_tasks` (עם סינון לפי כל שדה, כולל טווחי תאריכים), `add_task`, `update_task`, `delete_task`.
- **`agent_service.py`** — ה-agent. פונקציית `agent(query)` שולחת את ההודעה למודל Gemini יחד עם סכימת JSON שמתארת את ארבע הפונקציות. המודל מחליט אילו פונקציות להפעיל (ובאיזה סדר — למשל, לאתר משימה לפי כותרת לפני עדכון שלה), הפונקציות מופעלות בפועל, והמודל מתבקש לנסח מזה תשובה בשפה טבעית.
- **`main.py`** — שרת FastAPI עם `POST /chat` שמקבל `{"message": "..."}` ומחזיר `{"response": "..."}`.
- **`client/`** — אפליקציית React (בונוס) עם ממשק צ'אט בעברית ו-RTL, שמתקשרת עם השרת.

## הרצה

### 1. שרת (Python)

```powershell
.\venv\Scripts\Activate.ps1
python -m uvicorn main:app --reload
```

השרת יעלה על `http://127.0.0.1:8000`. תיעוד ה-API האינטראקטיבי זמין ב-`http://127.0.0.1:8000/docs`.

> אם מקבלים שגיאת "address already in use" — יש כבר תהליך שרץ על הפורט. סגרו אותו, או הריצו על פורט אחר עם `--port 8001` (ועדכנו בהתאם את `VITE_API_URL` בקובץ `client/.env`).

### 2. מפתח API

צריך מפתח מ-[Google AI Studio](https://aistudio.google.com/) בקובץ `.env` בשורש הפרויקט:
```
GOOGLE_API_KEY=your-key-here
```

⚠️ **מכסה חינמית**: מפתח חדש מקבל ביום הראשון מכסה יומית מצומצמת (כ-20 בקשות ל-`gemini-2.5-flash`). היא מתרחבת אוטומטית תוך יום-יומיים, או שאפשר להפעיל billing ב-Google AI Studio לקבלת מכסה גבוהה בהרבה.

### 3. קליינט (React, בונוס)

בטרמינל נפרד:
```powershell
cd client
npm install   # פעם ראשונה בלבד
npm run dev
```

ייפתח קישור (בד"כ `http://localhost:5173`) — זו אפליקציית הצ'אט.

### בדיקה ישירה של ה-API (Postman / curl)

```
POST http://127.0.0.1:8000/chat
Content-Type: application/json

{ "message": "תוסיפי לי משימה לקנות חלב מחר" }
```
